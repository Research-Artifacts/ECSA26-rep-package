# Eligibility Screening Criteria

## Inclusion Criteria

| ID | Criterion                                | Description                                                                                                                                                |
|----|------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------|
| I1 | Implementation of `Edge AI`-based system | The repository must contain an `Edge AI`-based system (application or framework) that utilizes AI on edge devices.                                         |
| I2 | Focus on performance for Edge Computing  | The project must show evidence of optimization for running on resource-constrained devices (e.g., model compression or edge-oriented inference libraries). |

## Exclusion Criteria

| ID | Criterion                                                    | Description                                                                                                                    |
|----|--------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------|
| E1 | Teaching material                                            | Repositories containing only tutorials, reports, workshops, or courses are excluded due to limited robustness as applications. |
| E2 | Dataset or pre-trained models                                | Repositories that provide only datasets or pre-trained models without a practical `Edge AI` application are excluded.          |
| E3 | Incomplete, obsolete, or archived                            | Projects that are incomplete, outdated, deprecated, or archived are excluded.                                                  |
| E4 | Code snippets or templates                                   | Repositories consisting only of code snippets, examples, or templates for other projects are excluded.                         |
| E5 | Demonstration or experimentation without practical objective | Projects exclusively focused on demonstrations or experimentation, without a practical objective, are excluded.                |
| E6 | Test artifacts                                               | Repositories containing only test scripts, test cases, or test artifacts are excluded.                                         |
| E7 | Encapsulated libraries or firmwares                          | Repositories that only encapsulate AI libraries (including edge-optimized libraries) or provide firmwares are excluded.        |
| E8 | Not an `Edge AI` project                                     | Repositories that do not implement an `Edge AI`-based system are excluded.                                                     |
| E9 | Toy projects                                                 | Repositories containing only toy or minimal example projects are excluded.                                                     |
